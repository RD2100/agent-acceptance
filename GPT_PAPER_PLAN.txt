第二层：论证强度，应该做

论证强度不是全文细读式评价，而是结构层面的强弱判断。比如：

YAML
argument_strength_check:

这一层很重要，因为你的论文题目不是一般政策解读，而是要形成学术论文的论证链。

第三层：引用质量，可以做，但先做“类型诊断”

YAML

  allowed_now:

    - "判断哪些部分不能只靠政策文件"
    - "判断是否需要补充高等教育规模、质量、结构、治理方面研究"

    - "逐条验证 CNKI 来源"
    - "完整参考文献清洗"
    - "全文引文一致性检查"

一句话：第一轮以结构和论证为主，同时判断引用支撑是否足够；不要一开始做完整引文审计。

## 4. 其他章节应该顺序处理，还是并行处理？

建议 **顺序处理**，不要并行。

```yaml
answer_4:
  recommended_mode: sequential
  parallel_processing: not_recommended_for_real_paper_first_run

原因是：这篇文章部分存在强递进关系：

YAML
paper_structure_logic:
  section_1: "必然逻辑：为什么要提质扩容"
  section_2: "内在张力：提质扩容过程中有什么矛盾"
  section_3: "平衡之道：如何处理质量与规模等关系"
  section_4: "协同治理：如何形成治理进路"

如果并行处理，容易出现三个问题：

YAML
parallel_risks:
  - "各节被孤立评价，看不出前后递进是否成立"
  - "后文治理建议可能脱离前文张力"
  - "多个 section packet 同时处理，增加隐私和范围失控风险"

建议顺序是：

YAML
recommended_sequence:
  phase_0:
    name: "摘要/引言 metadata preflight"
    goal: "确认全文问题意识、核心论点和结构承诺"

  phase_1:
    name: "第一节：必然逻辑"
    goal: "检查全文论证起点是否成立"

  phase_2:
    name: "第二节：内在张力"
    goal: "检查问题分析是否有解释力"

  phase_3:
    name: "第三节：平衡之道"
    goal: "检查调适逻辑是否回应张力"

  phase_4:
    name: "第四节：协同治理"
    goal: "检查治理进路是否回应前文问题"

  phase_5:
    name: "整体结构回看"
    goal: "检查标题、摘要、引言、四节之间是否一致"

每一节都要有一个 gate：

YAML
section_gate:
  before_next_section:
    - "本节 structural diagnosis 完成"
    - "revision blueprint 完成"
    - "utility evaluation 完成"
    - "无全文泄露"
    - "无外部上传"
    - "用户确认输出有用"
最终建议
YAML
final_plan:
  start_with: "Abstract/Introduction metadata preflight"
  then_process: "Section 1: 高等教育提质扩容的必然逻辑"
  diagnosis_focus:
    primary: "结构逻辑"
    secondary: "论证强度"
    tertiary: "引用功能"
  process_mode: "sequential, not parallel"
  advance_rule: "一节通过质量与安全审查后，再进入下一节"

一句话：先做摘要/引言预检，再顺序处理第一节、第二节、第三节、第四节；每一节以结构逻辑和论证强度为主，同时判断引用支撑是否足够，但暂不做全文式引文审计。
