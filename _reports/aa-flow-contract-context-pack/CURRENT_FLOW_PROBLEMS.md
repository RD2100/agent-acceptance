# Current Flow Problems

> Documented structural issues observed across multiple rounds

## Problem Matrix

| # | Problem | Concrete Example | Why Structural (Not Script Bug) |
|---|---------|-----------------|-------------------------------|
| 1 | **ready_to_dispatch not consumed** | Round-3 GPT accepted → post-decision driver wrote `ready_to_dispatch` → agent stopped. Next task spec on disk but never executed. | No contract forces consumption of ready task specs. Missing: dispatcher contract with `should_execute_next` field. |
| 2 | **TaskSpec generated but not executed** | `S3_PHASE2_TASKSPEC.md` was generated by post-decision driver → no runner picked it up → automation halted | Missing: TASKSPEC schema that says "when terminal=false and task spec exists, runner MUST execute it" |

| 4 | **terminal=false still produces terminal report** | Agent writes a final summary report wrapping up the run, even though `terminal=false` and a next task spec is waiting | Missing: TERMINAL_STATE_POLICY that says "terminal=false → continue, do not output final report" |
| 5 | **Markdown used as source of truth** | Automation reads `FINAL_REPORT.md` or `REVIEWER_INDEX.md` to decide whether to continue, instead of reading `FLOW_OUTCOME.json` | Missing: machine-readable schema as the single source of automation truth |

| 8 | **human_required taxonomy absent** | When GPT returns `human_required`, there's no structured categorization of WHY (destructive action? missing baseline? scope expansion?) | Missing: HUMAN_REQUIRED_TAXONOMY that maps reasons to required human actions |

## Root Cause Summary

The framework has excellent operational scripts (`oracle_chatgpt_cdp_handoff.py`, `oracle_gpt_reply_monitor.py`, `oracle_post_decision_driver.py`, `oracle_flow_state.py`, `oracle_decision_dispatcher.py`) but lacks:

1. A **contract layer** that defines what each state means
2. A **policy layer** that defines what to do when `terminal=false`
3. A **schema layer** that enables machine-reading of outcomes
4. A **gate layer** that enforces the distinction between transport/acceptance/dispatch

These gaps cause the same failure pattern: automation runs, GPT responds, task spec is generated, then the system stops because no contract says "you must continue."
