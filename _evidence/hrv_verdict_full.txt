BLOCKING-01：validate_hook_output.py 仍是旧语义

这是最大问题。

Rules (v2.2.0):

  - ai-guard exit_code != 0 (with sadp-audit pass) → overall_result must be PASS_WITH_WARNINGS
  - all exit_code == 0 → overall_result must be PASS

并且实际代码仍然检查：

这直接违背本任务目标。现在 hook v2.3.0 的目标是：

ai_guard_exit_1: BLOCKED
test_governance_exit_1: BLOCKED
PASS_WITH_WARNINGS: removed

更严重的是，validator 遇到 semantic warnings 后仍然：

也就是说，它可能发现语义不一致，但仍返回成功。这不能作为 fail-closed validator 接受。

BLOCKING-05：ai_guard exit code 捕获方式不可靠

pre-commit.governance.ps1 现在用 PowerShell Job 执行 ai_guard.py，但没有可靠捕获 Python 进程的真实 exit code，而是通过输出文本判断：

if ($aiGuardOutput -match "(?i)\bFAIL\b") {
    $aiGuardExit = 1
} else {
    $aiGuardExit = 0
}

这不是严格的 fail-closed 语义。如果 ai_guard.py 以非零退出但没有输出 “FAIL”，或者输出格式改变，hook 可能误判为 PASS。

建议后续改为可靠捕获子进程 exit code，不能依赖字符串匹配。

裁定：OPEN BLOCKER。
