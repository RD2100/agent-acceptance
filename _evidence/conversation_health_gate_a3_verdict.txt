2. pre_commit_health_advisory.py fail-graceful 语义核验
结论：部分通过，但内部错误语义不一致

用户提交说明写的是：

fail-graceful: 内部错误时 exit 0（advisory 不 block）

脚本顶部说明也写了类似语义：

Fail-graceful: any internal error → exit 0 with diagnostic message

但实际实现中，run_advisory() 对普通异常是：

Python
运行
except Exception as exc:
    ...
    return EXIT_MODULE_ERROR, result

而：

Python
运行
EXIT_MODULE_ERROR = 3

随后 main() 会执行：

Python
运行
sys.exit(exit_code)

这意味着某些内部错误会使脚本 CLI 退出码为 3，而不是 0。

虽然 pre-commit hook 的 overall_result 不会因此 BLOCK，因为 conversation-health stage 是 advisory；但这仍然和“fail-graceful: internal error exit 0”的设计声明不一致。

需要二选一修正：

方案一：保留 CLI exit 3，但文档和提交说明改为：
“advisory stage may return nonzero diagnostic exit codes; hook records but never blocks。”

方案二：真正 fail-graceful：
run_advisory() 的普通异常也返回 EXIT_HEALTHY = 0，同时在 result 中记录 advisory_error。

我更建议方案一，因为当前设计“非零 exit code 作为 advisory signal”是合理的，但必须写清楚：脚本可以非零，hook 不 block。
