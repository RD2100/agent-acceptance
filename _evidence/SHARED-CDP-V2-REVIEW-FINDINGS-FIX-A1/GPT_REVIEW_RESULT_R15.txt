1. 是否满足 SADP 0.R.2？

结论：不完全满足。

SADP 0.R.2 要求每次 @go run 产生 run evidence directory，至少包括：

diff.patch
test-output.md
safety-report.json
chain-evidence.json
review.md
review.yaml
final-report.md

我在 ZIP 中按精确文件名检索，这些文件均不存在。当前包提供的是：

DIFF_PATCH.txt
FULL_SUITE_OUTPUT_R15.txt
DRY_RUN_DISPATCH_10_CURRENT.json
MANIFEST.md

这属于“补充性证据归档”，不是 SADP 0.R.2 意义上的标准 run evidence directory。

更关键的是，提交说明中的复现命令不能按原样通过：

Bash
python -m pytest tests/ -v --tb=short

在解压目录直接运行后，测试收集阶段即失败，原因是测试文件导入 multi_project_router、dry_run_dispatch_10、sadp_pre_task_enforcer 等模块时找不到对应模块。即使我加上：

Bash
PYTHONPATH=source

也只能收集到 252 个测试，并出现 58 个失败，因为测试期待真实仓库结构中的 .agent/、.sadp/、scripts/、multi_cdp_launcher.py 等路径和文件。

因此，FULL_SUITE_OUTPUT_R15.txt 中的：

1038 passed, 0 failed, 21 warnings

只能证明它曾在 D:\agent-acceptance 的某个 Windows repo 环境中运行通过，不能证明这个 ZIP 本身是可独立复现的证据包。

另外，DRY_RUN_DISPATCH_10_CURRENT.json 文件名是 JSON，但内容在 JSON 结束后追加了：

--- DRY-RUN SUMMARY ---
...

所以它不是合法的纯 JSON 文件。这个问题不一定影响人工阅读，但会影响 evidence linter、CI 或 reviewer tooling 的机器校验。

3. 后续提交还需要什么证据？

后续每一轮最好恢复到严格的 evidence pack 结构。建议最低标准如下：

_evidence/<TASK_ID>/
├── diff.patch
├── test-output.md
├── safety-report.json
├── chain-evidence.json
├── review.md
├── review.yaml
├── final-report.md
├── commands.md
├── MANIFEST.json
└── hashes.sha256

其中要特别注意：

diff.patch 应该是 git diff --binary 或至少完整 git diff，不能只放部分文件。
test-output.md 应包含命令、cwd、Python/PowerShell 版本、exit code 和完整输出。
safety-report.json 必须是机器可读 JSON。
chain-evidence.json 要记录 executor、reviewer、task_id、input artifacts、output artifacts、hashes。
review.yaml 要有明确 verdict、blocking_findings、accepted_components、limitations。
涉及 self-protecting 文件时，必须提供 human_required decision record。
所有 .json 文件必须是合法 JSON，不能在 JSON 后附加人工摘要文本。
ZIP 内路径要尽量保留 repo 相对路径，例如 scripts/、.sadp/、docs/agent-runtime/，不要改放到 source/、governance/ 后再要求原命令复现。

建议新增一个一键复现脚本：

Bash
python tools/verify_evidence_pack.py _evidence/<TASK_ID>

它至少检查：

required files exist
all JSON parseable
diff.patch non-empty
test-output has exit code 0
review.yaml verdict consistent
chain-evidence references existing files
no summary-only submission
protected/self-protecting mutations have human_required record
