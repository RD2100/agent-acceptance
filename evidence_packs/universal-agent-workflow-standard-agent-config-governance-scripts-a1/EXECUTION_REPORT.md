## Summary

This task adds deployable governance script scaffolding to `scripts/awsp_scaffold.py` as part of the AWSP v1.2.0 standard. Five governance scripts are generated as deployable stubs by `create_scaffold()`, and `validate_scaffold()` is enhanced with checks for governance script existence, CLI entry points, and core function presence. The goal is to ensure that every AWSP-compliant project ships with a complete set of governance enforcement scripts.

## Changes Made

### 1. scripts/awsp_scaffold.py

Two feature blocks were added:

- **Governance Script Templates in AWSP_TEMPLATES**: Five governance script templates were added to the `AWSP_TEMPLATES` dictionary. Each template is a complete Python script with:
  - Shebang line and module docstring
  - Standard imports (argparse, json, sys, pathlib)
  - Core business function with TODO markers for project-specific customization
  - `main()` function with argparse CLI
  - `if __name__ == "__main__"` entry point
  - Standard exit codes (0=pass, 1=blocked/invalid)

  The five scripts are:
  1. `scripts/startup_read_gate.py` -- Verifies agent completed required reads (core: `gate()`)
  2. `scripts/pre_gpt_review_gate.py` -- Gates evidence pack before CDP submission (core: `gate()`)
  3. `scripts/pre_task_gate.py` -- Validates task authorization and continuity (core: `gate()`)
  4. `scripts/state_machine_runtime.py` -- State transition guards (core: `check_transition()`)
  5. `scripts/human_decision_record.py` -- Decision record management (core: `validate_record()`)

- **Governance Script Validation in validate_scaffold()**: A new validation block checks each governance script for:
  - File existence
  - CLI entry point presence (`if __name__` in source)
  - Core function presence (`def {core_func}(` in source)
  Missing scripts or missing functions produce descriptive error messages.

### 2. tests/test_cross_project_scaffold.py

A new `TestGovernanceScriptsScaffold` class was added with 8 test methods covering:

- Script creation: all 5 scripts generated by `create_scaffold()`
- Syntax validity: `py_compile.compile()` passes for all generated scripts
- CLI entry point: `if __name__` present in each script
- Validation pass: `validate_scaffold()` passes on complete scaffold
- Validation fail (missing script): removing one script causes validation failure
- Validation fail (missing core function): overwriting a script without its core function causes failure
- Validation fail (missing CLI entry): overwriting a script without CLI entry causes failure
- Validation fail (all missing): removing all scripts reports all missing
