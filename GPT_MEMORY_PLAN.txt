GitHub
 仓库结构中有 .claude、hooks、scripts、pyproject.toml、uv.lock 等文件，适合以第三方 skill 形式 vendored 到项目内。

3. 通用第三方 Skill 接口

需要新增一个标准接口：

YAML
skill_interface:
  root: ".claude/skills/"
  registry: ".claude/skills/registry.json"
  installer: "scripts/install_skill.py"
  verifier: "scripts/verify_skill_install.py"
  hook_merger: "scripts/merge_claude_hooks.py"

  install_command: >
    python scripts/install_skill.py
    --url https://github.com/coleam00/claude-memory-compiler
    --name claude-memory-compiler
    --target .claude/skills
    --pin HEAD
    --mode project
    --register
    --no-overwrite-settings

关键规则：

YAML
skill_safety_rules:
  - "clone 后先登记为 installed_pending_review。"
  - "不得直接覆盖现有 .claude/settings.json。"
  - "hooks 必须 merge，不得 replace。"
  - "必须记录 commit SHA。"
  - "必须生成 lock 文件。"
  - "必须运行 skill 自带 lint / structural check。"
  - "必须由 GPT review accepted 后再切换为 active。"
