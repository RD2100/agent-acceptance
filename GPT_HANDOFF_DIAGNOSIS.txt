1. 自动化交接工作流是否需要重新设计？

需要。当前流程应判定为 blocked，需要重构。

根因不是某一次 CDP 操作失败，而是流程设计把三种角色混在了一起：

YAML
roles_confused:
  Web_GPT:
    correct_role: "审查者、裁决者、计划生成者"
    wrong_role: "本地代码执行者"

  Claude_Code_or_local_agent:
    correct_role: "读取项目文件、执行 git、运行测试、生成 evidence pack、提交代码"
    wrong_role: "等待 Web GPT 执行本地仓库操作"

  CDP_automation:
    correct_role: "辅助提交 review prompt / 捕获 GPT 回复"
    wrong_role: "承担项目状态迁移和执行控制的核心通道"

所以这次失败的核心教训是：handoff 不是把 Web GPT 变成执行 agent，而是把本地执行 agent 的上下文恢复到可继续工作的状态。

2. 交接文档格式和提交方式应该是什么？

后续不应再依赖“长篇 HANDOFF 文本塞进 Web GPT 对话框”作为主流程。

推荐改成三层交接：

YAML
handoff_artifacts:
  primary:
    file: BOOT_CONTEXT.md
    purpose: "新 agent 冷启动入口，短上下文，优先读取"

  secondary:
    file: PROJECT_HISTORY.md
    purpose: "项目活蓝本，长期真相源"

  memory_layer:
    files:
      - memory/index.md
      - memory/tasks/*.md
      - memory/knowledge/*.md
    purpose: "按需读取任务生命周期和主题知识"

  optional_snapshot:
    file: HANDOFF_SNAPSHOT.md
    purpose: "仅在人类手动开新对话时使用，不作为自动化主路径"

提交方式应改成：

YAML
handoff_submission_method:
  for_local_agent:
    method: "文件路径 / 仓库文件"
    action: "Claude Code 直接读取 BOOT_CONTEXT.md + PROJECT_HISTORY.md + memory/index.md"
    no_cdp_required: true

  for_web_gpt:
    method: "只提交审查请求和 evidence pack"
    action: "Web GPT 只做 review，不接管执行"
    no_repo_execution_expected: true

交接成功标准也要明确：

YAML
handoff_success_criteria:
  local_agent:
    - "能读取 BOOT_CONTEXT.md"
    - "能读取 PROJECT_HISTORY.md"
    - "能定位当前任务 CONTEXT-COMPRESSION-A1-BINDING"
    - "能运行 git status / git diff --cached"
    - "能继续执行本地命令"

  web_gpt:
    - "只需回复 handoff_understood=yes"
    - "只确认当前状态和下一步计划"
    - "不应被要求执行 git 操作"
3. 新对话角色应该是谁？

新执行者应该是 Claude Code / 本地 Agent，不是 Web GPT。

YAML
correct_next_conversation:
  executor: "Claude Code / local coding agent"
  reviewer: "Web GPT"
  bridge: "CDP automation only for GPT review submission/capture"

Web GPT 的正确用途是：

YAML
web_gpt_should:
  - "审查 evidence pack"
  - "给 accepted / blocked 判决"
  - "给下一步授权"
  - "诊断流程设计问题"
  - "生成任务计划"

web_gpt_should_not:
  - "执行 git commit"
  - "访问本地仓库"
  - "修改文件"
  - "运行 pytest"
  - "替代 Claude Code 继续编码"

因此，这次“新对话 GPT 无法执行 git 操作”不是 GPT 失败，而是流程把任务交给了错误角色。
